# FEDRI Framework  
**Fractal • Experiment • Distillation • Recursion • Integration**  
A fractal cognitive engine where agents evolve through iterative simulation, behavioural distillation, emotional recursion, and continuous integration of experience.

---

## FEDRI as an Agent‑Based Cognitive Simulation Engine

FEDRI is an agent‑based cognitive simulation engine that models how autonomous psychological systems perceive, interpret, react, and evolve over time. It integrates deep psychological modelling, multi‑agent interaction, behavioural distillation, recursive state evolution, and integrative learning into a unified architecture.

Instead of scripting behaviour, FEDRI generates agents whose actions emerge from their internal cognition, emotional regulation (via CIR), contextual pressure, relational dynamics, and resource constraints. The result is a system where behaviour is not written — it is discovered through simulation.

FEDRI is designed for environments where behaviour must remain coherent, predictable, and psychologically realistic, even under stress, conflict, or shifting context. Agents adapt, destabilise, stabilise, and evolve as their internal states change, producing emergent interactions that can be analysed, repeated, or extended across simulations.

---

## Origin
FEDRI originated from my transition from emergency‑medicine training into game development, where I became deeply interested in why some characters feel psychologically real and why some narratives create immersion. I began analysing character construction across games, books and films, and later built my own simulations to understand how behaviour emerges when characters are treated as autonomous psychological systems rather than scripted entities. Over time, iterative simulation, behavioural distillation and integration with CIR produced a scalable cognitive engine capable of modelling both fictional agents and real users. FEDRI evolved through narrative experimentation, psychological modelling and real‑world behavioural analysis into the agent‑based architecture it is today.

---

## How FEDRI Works

### Fractal  
Understanding the psyche as a fractal structure that can be explored at multiple depths. Each agent contains:

- a five‑level emotional stability model (CIR)  
- layered defences, motivations, and impulses  
- behavioural patterns that only emerge under interaction  

Fractality means the psyche is not a list of traits but a recursive structure that reveals deeper layers under pressure, context, or relational triggers.

### Experiment  
Running agents through simulated scenarios to observe how they interpret, react, and adapt. Instead of writing a scene, an experiment is run where:

- the agent interprets the situation  
- generates emotion, thought, and impulse  
- performs an action  
- influences other agents  

Scenes are not predetermined — they emerge from the interaction of autonomous psychological systems.

### Distillation  
Extracting the meaningful behavioural signals from the simulation:

- key emotional reactions  
- regulatory patterns  
- relational dynamics  
- interpretation loops  
- behavioural signatures  

Distillation separates noise from signal and produces a predictive behavioural model that can be used in subsequent iterations.

### Recursion  
Every scene changes the agent. Every change in the agent changes the next scene. This creates a spiral dynamic, where:

- CIR shifts upward or downward  
- masks strengthen or crack  
- relationships stabilise or destabilise  
- behaviours become increasingly consistent  

Recursion ensures the agent is not static — it evolves through experience.

### Integration  
Incorporating distilled insights back into the agent’s psyche:

- updating fears  
- updating motivations  
- updating boundaries  
- updating interpretations  
- updating defensive patterns  

The agent learns, remembers, and adapts — producing long‑term behavioural coherence.

## FEDRI Operational Pipeline

1. **Initialize Agent (Fractal Layer)**  
   Define goals, fears, constraints, resources, relational context.  
   Initialize CIR baseline.  
   Initialize fractal depth of psyche.

2. **Initialize Scene (Experiment Layer)**  
   Define environment, stakes, timing, social context.  
   Place agents in the scene.

3. **Perception Pass**  
   Each agent generates a subjective interpretation of the scene.

4. **Reaction Pass**  
   Each agent generates emotion, thought, impulse, action, mask.

5. **Stimulus Injection**  
   When the system stabilizes, introduce a new stimulus: environmental, relational, informational.

6. **Multi‑Agent Loop (Recursion Layer)**  
   Agents react to each other. CIR shifts. Dynamics evolve. New behaviours emerge.

7. **Distillation Pass**  
   Extract meaningful patterns: behavioural signatures, emotional signatures, relational signatures.

8. **Integration Pass**  
   Update agent psyche with distilled insights. Adjust CIR baseline if needed. Prepare agent for next simulation.

# Three‑Resolution Simulation in FEDRI

FEDRI is a fractal cognitive simulation engine operating at multiple levels of resolution. It models human behaviour with high psychological fidelity when needed, while remaining efficient and scalable for real‑world R&D workflows.

**High‑Resolution • Mid‑Resolution • Low‑Resolution**  
**CIR‑Grounded • SPINE‑Contextualised**

The three resolutions correspond to different use cases, different cognitive depths and different computational demands. Together, they form a coherent continuum that allows FEDRI to move seamlessly from immersive psychological exploration to fast, operational decision support.

All three resolutions operate on the same underlying agent architecture, allowing simulations to scale up or down without losing psychological coherence.

---

## High‑Resolution Simulation — Immersive Cognitive Modelling

High‑resolution FEDRI simulations operate at the level of micro‑thoughts, micro‑reactions and moment‑to‑moment emotional dynamics. These simulations are typically expressed through immersive narrative sequences (synthetic data) that reveal internal monologue and emotional shifts, sensory and cognitive micro‑signals, conflict between impulses and regulation, subtle behavioural cues, dynamic changes in stability (CIR levels), and the full trajectory of a psychological state.

High‑resolution simulations also function as a controlled synthetic‑data generator for testing cognitive and emotional mechanisms.

### Expressive Power — What High‑Resolution Shows
This resolution demonstrates the maximum expressive power of FEDRI. It is used to validate the internal coherence of the model, explore edge cases and extreme states, test how agents behave under stress or dysregulation, and refine the emotional and cognitive mechanics of the system.

### Role in Workflow — Laboratory Environment
High‑resolution simulations are not used directly in product work. Instead, they serve as the laboratory environment where the architecture is stress‑tested and calibrated before being compressed into more operational forms.

---

## Mid‑Resolution Simulation — Predictive User Modelling

Mid‑resolution FEDRI simulations are the core of my R&D practice and form the predictive modelling layer. They model emotional load, cognitive constraints, behavioural tendencies, decision‑making patterns, ND‑specific processing styles, avoidance loops and motivational dynamics.

### Integration — Where Mid‑Resolution Lives
This resolution is used in:

- SPINE Stage 2 — Cognitive Modelling  
- SPINE Stage 3 — Behavioural Friction Mapping  
- SPINE Stage 4 — Hypothesis Architecture  
- MAKO — behavioural layer  
- MAZE — cognitive architecture  

### Operational Mode — Primary Working Resolution
Mid‑resolution simulations are fast enough for iterative design, detailed enough to predict behaviour, structured enough to support intervention design, and grounded enough to integrate with real‑world constraints. This is the primary operational mode of FEDRI.

---

## Low‑Resolution Simulation — Rapid Insight Engine

Low‑resolution FEDRI simulations are used for fast, on‑demand reasoning during strategy sessions, rapid prototyping, early‑stage hypothesis generation and quick behavioural predictions. They operate on simplified emotional vectors, high‑level behavioural patterns, coarse‑grained cognitive states and quick scenario sketches.

### Speed — Agile Insight Layer
This resolution trades depth for speed, enabling 10–15 minute insight cycles, rapid comparison of user variants, quick identification of friction points and fast behavioural forecasting.

### Usage — On‑Demand Reasoning
Low‑resolution simulations act as the agile layer of the architecture, supporting real‑time discussions and early decision‑making without sacrificing psychological coherence.

# FEDRI Origin

FEDRI originated at the intersection of two seemingly unrelated domains: emergency‑medicine training and game development. After transitioning from medical studies into the game industry, I became increasingly interested in why some fictional characters feel psychologically real while others remain flat. This led me to analyse narrative structures across games, books and films, focusing on works where the psychology of characters felt coherent, consistent and believable.

As I began constructing my own narratives, I noticed a clear pattern: when writing immersively, characters behaved as if they had their own internal logic. I did not need to decide their actions — I placed them in a scene, and their reactions, conflicts and development emerged naturally. When writing technically, the result was correct but lacked depth. This distinction became the seed of FEDRI: the idea that characters should not be written, but *simulated*.

Around 2015 I began creating detailed character sheets as an early form of documentation. These sheets evolved into iterative simulations: I generated scenes, observed how characters interacted, extracted behavioural patterns, and fed the insights back into the model. Over time, FEDRI combined with CIR to produce more than 10,000 pages of immersive narrative data — a long‑form synthetic dataset through which the system refined itself and expanded its psychological mechanics. In parallel, I continued mapping films, books and games, integrating their structural patterns into the model.

A turning point came when I realised that the same mechanisms that produced psychologically coherent fictional agents could also model real people. FEDRI + CIR began to map user behaviour in IT projects with surprising accuracy. Although I never had the opportunity to build the narrative‑driven game I envisioned — the complexity of the system exceeded what could be implemented at the time — the architecture found a different application.

In 2017 I received a grant to implement an innovative motivational system in a school environment. FEDRI and CIR were tested in real‑world conditions with students, under strict constraints and limited resources. This project validated the behavioural and regulatory mechanics of the model and led to the creation of MAKO and the early foundations of SPINE.

From 2018 onward, I focused on developing FEDRI and CIR as tools for modelling real users, predicting behaviour and analysing interpersonal dynamics. Narrative simulation remained part of the workflow, serving as a controlled environment for expanding the architecture.

In 2024 I began studying biochemistry and neurochemistry, which significantly enriched both FEDRI and CIR by grounding their mechanisms in neurobiological processes. In 2025 I applied the system to behavioural forensics in a company struggling with persistent interpersonal conflict. FEDRI + CIR produced a mechanistic remapping of team dynamics that allowed participants to understand each other’s patterns, resolving approximately 90% of the conflict after a single analysis.

Today, FEDRI is a mature agent‑based cognitive simulation engine: a system that grew from narrative experimentation, psychological modelling, real‑world behavioural analysis and continuous integration of insights across multiple domains.

# Example 01 — Stress, Hypervigilance & Social Regulation  
*A high‑resolution simulation of acute stress, micro‑reactions, emotional inhibition and CIR‑level shifts during a high‑pressure scenario.*

---

## High‑Resolution Slice — Agent 1  
**Scenario (compressed):**  
Agent 1 has spent an hour explaining a complex financial product to a client who ultimately declines. The interaction is formally polite but emotionally frustrating.

**Initial state (simplified subset):**  
- Physical energy: 3/5  
- Cognitive energy: 2/5  
- Social energy: 3/5  
- Apathy–Engagement: 2/5  
- Friendliness–Aggression: 2/5  
- Sadness–Satisfaction: 2/5  
- Need for safety: 2/5  
- Need for recognition: 3/5  
- Need to be understood: 3/5  
- Need for control: 3/5  
- Cortisol: 3/5  
- Adrenaline: 2/5  
- Oxytocin: 1/5  

**Sample update rules (simplified):**  
- Cortisol ↑ → short‑term physical energy ↑, long‑term fatigue ↑  
- Cortisol ↑ → satisfaction ↓, apathy ↑  
- Need for control ↑ → adrenaline ↑  
- Adrenaline ↑ → friendliness–aggression shifts toward aggression  

**Event:**  
The client declines the product after a long explanation. Agent 1 maintains a professional smile but shows a micro‑expression of irritation.

**State update (qualitative):**  
- Cortisol: 3/5 → 4/5  
- Cognitive energy: 2/5 → 1/5  
- Need for recognition: 3/5 → 4/5  
- Need for control: 3/5 → 4/5  
- Adrenaline: 2/5 → 3/5  
- Friendliness–Aggression: 2/5 → 3/5  
- Sadness–Satisfaction: 2/5 → 1/5  

**Behavioural interpretation:**  
Outward behaviour remains polite and regulated, but internal vectors shift toward tension and future sensitivity to stressors. FEDRI tracks these micro‑updates step by step, enabling high‑resolution modelling of when and why regulation weakens — and how it evolves across simulations.

---

## Mid‑Resolution Slice — Agent 1  
*(Emotionally frustrating but socially regulated situation)*

**Scenario:**  
Agent 1 invests significant cognitive effort into a long client interaction that ends with rejection. The agent maintains professionalism but experiences internal irritation and depletion.

**State overview:**  
- Moderate physical fatigue  
- Low cognitive resources; attention narrowing  
- Emotional state: suppressed irritation, mild frustration  
- Social strategy: maintain professionalism, avoid conflict  
- Needs activated: recognition, control, competence  
- Biochemical profile: elevated cortisol, mild adrenaline increase  
- Regulation: stable but strained (CIR ~3)  

**Key dynamics:**  
- Effort–reward mismatch increases frustration and cortisol.  
- Suppressed irritation raises internal tension without breaking the social mask.  
- Need for recognition rises due to lack of acknowledgement.  
- Need for control rises due to outcome mismatch.  
- Cognitive fatigue reduces tolerance for further demands.  

**State shift:**  
- Cortisol moderately elevated → emotional tension  
- Cognitive energy reduced → lower resilience  
- Social mask intact → behaviour remains polite  
- Internal dissatisfaction increases → future sensitivity rises  

---

## Low‑Resolution Slice — Agent 1  
*(Frustration under social regulation)*

**Scenario:**  
Agent 1 invests significant effort into a client interaction that ends with rejection. Outward behaviour remains professional, but the internal state shifts toward frustration and depletion.

**Core state shift:**  
- Emotional tone moves from neutral to mildly negative.  
- Cognitive resources drop due to perceived wasted effort.  
- Recognition and control needs intensify.  
- Cortisol rises, increasing tension but not breaking regulation.  
- Social mask remains intact; behaviour stays polite.  

**Overall interpretation:**  
The agent experiences a controlled, low‑grade stress response. Regulation holds, but internal strain increases, making the agent more sensitive to subsequent pressure.

# Example 02 — Social Pressure, Masking & Cognitive Load  
*A high‑resolution simulation of social threat perception, masking, micro‑lying, emotional dissonance and multi‑agent modelling in a workplace‑adjacent context.*

---

## High‑Resolution Slice — Agent 1  
**Scenario (compressed):**  
Agent 1 is in a confined space inside a stationary vehicle. Agent 2 has stepped outside and has not returned for longer than expected. The silence and lack of information are interpreted as potential signals of something being wrong. Ambient noises from outside are ambiguous and difficult to classify.

**Initial state (simplified subset):**  
- Physical energy: 2/5  
- Cognitive energy: 1/5  
- Social energy: 1/5  
- Apathy–Engagement: 4/5 (high alertness, scanning for cues)  
- Friendliness–Aggression: 1/5 (no aggression; tension‑driven caution)  
- Sadness–Satisfaction: 1/5 (low satisfaction, high unease)  
- Need for safety: 5/5  
- Need for control: 5/5  
- Need to be understood: 4/5  
- Cortisol: 5/5  
- Adrenaline: 4/5  
- Oxytocin: 1/5  

**Sample update rules (simplified):**  
- Cortisol ↑ → cognitive energy ↓, physical tension ↑  
- Adrenaline ↑ → startle response ↑  
- High safety need + low oxytocin → perceived risk ↑  
- High control need + high cortisol → high‑arousal appraisal  

**Event:**  
A thought appears: “Why is he taking so long?”  
A distant door slam from another vehicle is interpreted as a potentially relevant cue. The agent’s body responds with a sharp tension spike.

**State update (qualitative):**  
- Cortisol: 5/5 → 5/5 (remains elevated)  
- Adrenaline: 4/5 → 5/5 (startle + uncertainty)  
- Physical energy: 2/5 → 1/5 (tension increases, initiative drops)  
- Cognitive energy: 1/5 → 0/5 (narrowed focus, reduced evaluation capacity)  
- Need for safety: 5/5 → 5/5  
- Need for control: 5/5 → 5/5  
- Apathy–Engagement: 4/5 → 5/5 (full cue‑scanning mode)  

**Behavioural interpretation:**  
The agent remains physically still and highly focused on environmental cues. Ambiguous sounds are interpreted as potentially meaningful. FEDRI treats this as a high‑arousal misinterpretation loop, where internal vectors lock into a configuration that prioritises rapid threat detection over balanced evaluation.

---

## Mid‑Resolution Slice — Agent 1  
*(Heightened alertness, uncertainty, high‑arousal appraisal)*

**Scenario:**  
Agent 1 is waiting inside a vehicle while Agent 2 is outside. The absence of updates and the surrounding silence are interpreted as potential signals of risk. The agent becomes increasingly alert and reacts strongly to ambiguous sounds.

**State overview:**  
- Physical state: tension, readiness to react  
- Cognitive state: low capacity, intrusive “what‑if” thoughts  
- Emotional state: unease, anticipatory tension  
- Social state: Agent 2 functions as a stabilising reference point  
- Needs activated: safety (max), control (max), reassurance  
- Biochemical profile: high cortisol, high adrenaline, low oxytocin  
- Regulation: strained but intact (CIR ~4)  

**Key dynamics:**  
- Lack of information → elevated alertness.  
- Silence becomes a cue requiring interpretation.  
- “Why is he taking so long?” amplifies physiological arousal.  
- Startle response to external noise indicates high sensitivity to ambiguity.  
- Reduced cognitive energy → difficulty evaluating alternatives.  
- Presence of Agent 2 normally stabilises the system; absence removes that anchor.  

**State shift:**  
- Cortisol remains high → sustained tension  
- Adrenaline spikes → rapid reactivity  
- Cognitive energy drops → narrowed focus  
- Safety and control needs dominate → high‑arousal appraisal  
- Partial regulation returns when Agent 2 re‑enters the environment  

---

## Low‑Resolution Slice — Agent 1  
*(High‑arousal interpretation loop)*

**Scenario:**  
Agent 1 is alone in a confined space, waiting for Agent 2 to return. Silence and lack of information are interpreted as potential risk. A minor external sound triggers a strong orienting response.

**Core state shift:**  
- Emotional tone shifts toward tension and uncertainty.  
- Cognitive capacity narrows into “what‑if” thinking.  
- Safety and control needs dominate.  
- Cortisol and adrenaline remain high, driving alertness.  
- Regulation becomes effortful; the agent oscillates between stillness and rapid reactivity.  

**Overall interpretation:**  
The agent enters a high‑arousal state where perception becomes risk‑biased and decision‑making capacity narrows. Regulation remains possible but requires external stabilising cues, such as the return of Agent 2.
